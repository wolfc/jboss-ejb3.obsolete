Stateful Beans

It is also very easy to create a Stateful Bean with EJB 3.0 using deployment descriptors, as opposed to annotations. Take a look at ShoppingCartBean.java and the corresponding deployment descriptors ejb-jar.xml and jboss.xml

All bean types are homeless in EJB 3.0, so the home and local-home elements are not required. Only ejb-name, ejb-class, and session-type are required. The session-type element has the value Stateful.

ShoppingCartBean implements one interface. This is the remote interface of the EJB. There is no local interface specified. Currently, you can only have one remote and/or local interface.

Take a look at ShoppingCart.java. To define this as the remote interface of ShoppingCart bean you only need the remote element.

JNDI Bindings

The Calculator bean will have one JNDI bindings for the remote interface as specified in jboss.xml with the jndi-name element.

Client

Open up Client.java. You'll see that it looks up the stateful bean under its remote interface's fully qualified classname. Also notice that there is no Home interface and you can begin executing on the stateful bean right away. When you access the bean in JNDI, an instance of the stateful bean will be created on the server. So, when you need a different instance of the stateful bean, you do an additional jndi.lookup() to get this new reference.

Building and Running

To build and run the example, make sure you have ejb3.deployer installed in JBoss 4.0.x and have JBoss running. See the reference manual on how to install EJB 3.0. 
Unix:    $ export JBOSS_HOME=<where your jboss 4.0 distribution is>
Windows: $ set JBOSS_HOME=<where your jboss 4.0 distribution is>
$ ant
$ ant run

Jar structure

EJB 3.0 beans must be packaged in a JAR file with the suffix .ejb3. Running the ant script above creates a JAR file within the deploy/ directory of JBoss. All that needs to be in that jar is your server-side class files. So basically just the ShoppingCartBean and the interfaces it implements. JBoss will automatically browse the JAR file to determine if any EJBs are annotated by any classes within it. THere is no precompilation step.
